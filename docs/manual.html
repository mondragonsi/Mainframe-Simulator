O que é este simulador?

Este simulador reproduz fielmente o comportamento de um IBM z/OS Mainframe — o tipo de computador que processa bilhões de transações bancárias, aéreas e governamentais todos os dias. Você interage com uma interface TSO/ISPF no terminal, exatamente como faria em um mainframe de produção.

📷

IMS DB

Banco de dados hierárquico da IBM. Aprenda DL/I, DBDs, PSBs, PCBs e SSAs — a tecnologia por trás de centenas de bancos e seguradoras.

📁

z/OS Datasets

PS (sequencial), PDS (particionado) e VSAM (KSDS, ESDS, RRDS). O jeito mainframe de armazenar dados.

⚙

IDCAMS

Utilitário z/OS para criar, listar, excluir e copiar datasets. Cria clusters VSAM, aloca PS/PDS, exibe catálogos.

💻

ISPF Panels

Interface TSO/ISPF com PF Keys funcionais (PF1=Help, PF3=End, PF7=Up, PF8=Down, PF12=Cancel), exatamente como no z/OS real.

📡

IMS TM

Transaction Manager: regiões MPP (online) e BMP (batch). Execute transações online e programas em lote.

🏗

Address Spaces

Modelo MVS de espaços de endereçamento — monitore regiões IMS, DB2 e JES2 como um operador z/OS.

🔄 Exemplo Passo a Passo: Criar PDS, Editar Membro e Navegar

  • Abrir ISPF Utilities: No prompt principal, digite /DS e pressione Enter. O painel ISRUTIL será exibido.
  • Ir para Data Set Utility: Digite 2 em Option ===>. O painel ISRDUTIL abre. Digite o nome desejado no campo Data Set Name, ex: MINHA.FONTE.
  • Alocar o PDS: Digite A em Option ===>. O formulário ISRACDA abre com os parâmetros pré-preenchidos. Confirme o tipo PDS e pressione Enter para alocar.
  • Abrir Dslist: Pressione PF3 para voltar ao ISRUTIL, depois digite 4. No painel ISRDSAB, insira MINHA.* em Dsname Level e pressione Enter.
  • Ver membros do PDS: Na lista ISRDSLIST, coloque M na coluna Cmd ao lado de MINHA.FONTE. O painel ISRMLM exibe os membros.
  • Editar um membro: No ISRMLM, coloque E ao lado de um membro. O painel ISREDIT abre. Digite o conteúdo e pressione PF3 para salvar (STOW R).
  • Browse do membro: De volta ao ISRMLM, coloque B ao lado do membro. O painel ISRBRO exibe o conteúdo com numeração de linhas. Pressione PF3 para fechar.

📦 Tipos de Dataset z/OS

📄

PS — Physical Sequential

Arquivo sequencial. Lido do início ao fim. Equivale a arquivo Unix. ALLOC DSN PS

📂

PDS — Partitioned

Coleção de membros. Como diretório. Usado para JCL, COBOL, load modules. ALLOC DSN PDS

🔑

VSAM KSDS

Key-Sequenced. Acesso direto por chave. O mais comum — como índice de BD. DEFINE CLUSTER

📋

VSAM ESDS

Entry-Sequenced. Sem chave, acesso pela posição RBA. Bom para logs.

🔢

VSAM RRDS

Relative Record. Acesso por número de slot. Para tabelas de tamanho fixo.

📚

GDG — Generation Data Group

Família versionada: MY.DATA(0)=atual, MY.DATA(-1)=anterior. Backups rotativos.

💡
Nomenclatura: Qualificadores separados por ponto: DEPT.APP.TIPO. Máx 44 chars total, até 8 chars por qualificador. Sempre maiúsculas.

IMS DB — Banco de Dados Hierárquico

O IMS (Information Management System) da IBM usa um modelo hierárquico — como uma árvore. Dados são organizados em segmentos pai-filho. Exemplo: HOSPITAL → PACIENTE → DIAGNÓSTICO.

🌳 Hierarquia IMS — Banco HOSPITAL
  HOSPITAL  (DBD: HOSPITL)
  │
  ├── PACIENTE  (key: PATNUMBR)
  │       ├── DIAGNOSE
  │       └── TREATMNT
  └── DOCTOR    (key: DOCNUMBR)
          └── PATIENT (lógico)

🔐 Conceitos Fundamentais

📐

DBD — Database Definition

Define o esquema: segmentos, campos, tamanhos e relações hierárquicas pai/filho.

🔒

PSB — Program Spec Block

Define a visão do programa sobre os bancos. Contém um ou mais PCBs. Cada programa tem seu PSB.

📡

PCB — Program Comm Block

Regras de acesso de um programa a um DBD. Status code é retornado aqui após cada DL/I call.

🔍

SSA — Segment Search Argument

Filtro de busca: PACIENTE(PATNUMBR=12345). Navega a hierarquia e qualifica registros.

Glossário Mainframe

📷 IMS — Information Management System

Sistema de gerenciamento de banco de dados hierárquico da IBM, lançado em 1968. Ainda processa trilhões de transações diárias em bancos e companhias aéreas. Usa modelo hierárquico (árvore pai-filho), diferente do modelo relacional SQL.

📐 DBD — Database Definition

Define a estrutura física do banco IMS: segmentos, campos (nome/tipo/tamanho), hierarquia e tipos de acesso. É o equivalente ao CREATE TABLE do SQL.

🔒 PSB / PCB

O PSB é a visão do programa sobre os bancos — define quais bancos pode acessar. Contém um ou mais PCBs. O PCB define regras de acesso e é onde o IMS retorna o status code após cada DL/I call.

📞 DL/I — Data Language/I

Interface de chamada do IMS. COBOL chama: CALL 'CBLTDLI' USING function pcb io-area ssa. Funções: GU, GN, GNP, GHU (leitura), ISRT (inserção), DLET (exclusão), REPL (atualização).

🔍 SSA — Segment Search Argument

Filtro de busca DL/I. Formato: SEGNAME (CAMPO OP VALOR). Exemplo: PACIENTE(PATNUMBR =00000001). Operadores: =, <, >, <=, >=. Pode ser não-qualificado (só nome) ou qualificado (com critério).

📁 VSAM — Virtual Storage Access Method

Método de acesso z/OS para datasets de alto desempenho. Tipos: KSDS (por chave), ESDS (sequencial por posição), RRDS (por número de slot). Criados com IDCAMS DEFINE CLUSTER.

⚙ IDCAMS — Integrated Data Catalog Access Method Services

Utilitário z/OS para gerenciar datasets e catálogos ICF. Comandos: DEFINE CLUSTER, DELETE, LISTCAT, REPRO, PRINT, ALLOCATE.

📡 MPP / BMP — Regiões IMS

MPP: região online. Processa transações em tempo real. Ativado por mensagem de terminal. BMP: região batch. Mistura acesso a IMS DB com processamento batch — pode ler mensagens da fila ou processar arquivos.

💻 TSO/ISPF

TSO (Time Sharing Option): shell interativo do z/OS. ISPF (Interactive System Productivity Facility): interface de menus sobre o TSO. O simulador reproduz a aparência e comportamento do ISPF, incluindo PF keys e a linha COMMAND ===>.

Perguntas Frequentes

🔨 Como compilar no Windows?

Instale o MinGW-w64 (GCC para Windows) — recomendamos o w64devkit. Extraia e adicione ao PATH. Depois execute make na pasta c:\dev\Mainframe-Simulator.

📁 Onde os datasets ficam armazenados?

Os datasets são mantidos em memória durante a sessão. O catálogo ICF é inicializado a cada execução. Em versão futura, os datasets serão persistidos em arquivos locais entre sessões.

⌨ As PF Keys não funcionam no meu terminal

O simulador detecta teclas F1-F12 via sequências de escape ANSI. Se não funcionar, use: END (=PF3), Q (sair). O Windows Terminal e Git Bash normalmente funcionam bem.

📷 Como carregar o banco HOSPITAL?

Execute make demo. Isso inicia o simulador com o banco HOSPITAL pré-carregado e executa uma sequência de DL/I calls demonstrando os segmentos PACIENTE, DOCTOR, DIAGNOSE etc.

📊 Como ver os status codes IMS?

Após cada DL/I call, o PCB retorna um status code de 2 caracteres. Brancos = sucesso. Os mais comuns: GE (não encontrado), GB (fim do banco), II (chave duplicada).

🚀 Quais recursos estão planejados?

Roadmap: DB2 (SQL embarcado, SQLCA, cursores), COBOL Runtime (interpretador com CBLTDLI), JCL Interpreter (submissão de jobs, JES2), e persistência de datasets entre sessões.
